--- Требовния к инструментам:

- GrammyJS для Telegram bot
- dotenv для env переменных
- Prisma для работы с postgres и моделями
- i18n для работы с locale

---

Структура проекта:

- В папке app/ лежат нужный функционал для запуска приложения и также middleware, bd config, locale
- В папке handlers/ уже разбитые на нужные сущности части кода которые каждый отвечает за свою часть
- В папке handlers/auth уже есть готовая структура которую тебе нужно придерживаться:
  - auth.const.ts => Константы которые используются только в домене auth
  - auth.handlers.ts => Handler'ы для команд бота которые используются только в домене auth
  - auth.service.ts => Работа с базой данных которая используются только в домене auth
  - auth.types.ts => Типы которые используются только в домене auth
  - auth.entry.ts => public api этого домена
- В папке shared/ переиспользуемые куски кода, переменные, типы, логика итд
- Поток зависимостей между слоями таков shared самый низкий слой, он может импортироваться в любой выше стоящий, app самый высший слой, из него в другие слои ничего импортировать нельзя, handlers средний слой он может импортировать слой shared но не app, и может быть импортирован в app
